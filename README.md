# Sonalia Public Site

Esta carpeta contiene solo los archivos publicos que debes subir a un repositorio nuevo y publico para GitHub Pages.

## Archivos incluidos

- `index.html`
- `privacy.html`
- `terms.html`
- `.nojekyll`

## Recomendacion

Crea un repositorio nuevo, por ejemplo `sonalia-site`, y sube solo el contenido de esta carpeta.

## Pasos

1. Crea un repositorio nuevo publico en GitHub.
2. Sube unicamente los archivos dentro de `site/`.
3. En GitHub, entra a `Settings` > `Pages`.
4. En `Build and deployment`, elige `Deploy from a branch`.
5. Selecciona la rama principal del repo nuevo y la carpeta `/ (root)`.
6. Espera a que GitHub Pages publique el sitio.

## URLs esperadas

Si el repo nuevo se llama `sonalia-site` y el usuario es `Dev-Roxe`, las URLs quedaran asi:

- `https://dev-roxe.github.io/sonalia-site/`
- `https://dev-roxe.github.io/sonalia-site/privacy.html`
- `https://dev-roxe.github.io/sonalia-site/terms.html`

## Campos para Google Cloud

- Pagina principal: URL de `index.html`
- Politica de privacidad: URL de `privacy.html`
- Terminos del servicio: URL de `terms.html`
- Dominio autorizado: `github.io`

## Nota

No publiques el repositorio principal de la app para esto. Usa un repo nuevo y limpio solo con estos archivos.
